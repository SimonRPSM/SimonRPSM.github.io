## I Don't Know What Anyone is Talking About
The world of Software is filled with pretty arcane terminology that can be pretty daunting the non initiated. I would like to taxonomize this lexicon in the following pieces:

1. Computer Science Terminology
2. Software Engineering methodologies and concepts
3. Software Engineering Methodologies (bear with me)
4. Internal company jargon
5. Specific frameworks, tools, etc

The first and fourth items are reasonably self-explanatory. You can learn the lexicon of computer science by ... learning computer science. This will mean that when someone mentions implementing a Depth-first search algorithm for something you'll nod and say that it sounds like a good idea because it actually does, not because you dont want to say that you know "some of these words".

Every company creates a vast amount of internal terminology. Servers have names, libraries have names, internal methodologies and processes have names, etc.

Things get a bit more interesting in attempting to separate items two and three.

There are some Methodologies with a capital M that attempt to describe the holistic process (or pieces of this process) of creating software. For example, if someone mentions the *Dynamic Systems Development Method*, it refers to a specific way to develop software that's an attempt to mix waterfall and agile methodologies (I'm oversimplifying that).

This is slightly different than concepts or methodologies like the entirety of the book *Clean Code*, for example, that describes a whole plethora of things that are not quite a "system" but a series of recommendations on how to build software. Similarly, SOLID is a conglomeration of design principles that tell you how to think about writing the software, but they are not really a Methodology.

The last item is the plethora of tools that are out there. Each industry or application will have some tools that are ubiquituous and known to work together. For example, if you are working with Clojure, then Leiningen is a thing you'll need to be familiar with. If you are using LabVIEW, then you'll likely use VIPM.
