I write code and use a compiler. It does magical stuff.

Code is portable (sometimes) and sometimes it isn't. Sometimes there's "VMs" and sometimes there ain't.

I know that code becomes "bits" at some point, but how does that work?

What things are worthwhile reading? Charles Petzold's Code, NAND2Tetris. Former for a holistic introduction, the later for actually building all this stuff.
